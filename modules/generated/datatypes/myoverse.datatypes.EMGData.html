class myoverse.datatypes.EMGData(input_data, sampling_frequency, grid_layouts=None)[source]#

Class for storing EMG data.

Parameters:

  • input_data (np.ndarray) â

    The raw EMG data. The shape of the array should be (n_channels, n_samples) or (n_chunks, n_channels, n_samples).#

    Important

    The class will only accept 2D or 3D arrays.

    There is no way to check if you actually have it in (n_chunks, n_samples) or (n_chunks, n_channels, n_samples) format. Please make sure to provide the correct shape of the data.

  • sampling_frequency (float) â The sampling frequency of the EMG data.

  • grid_layouts (Optional[List[np.ndarray]], optional) â

    List of 2D arrays specifying the exact electrode arrangement for each grid. Each array element contains the electrode index (0-based).

    Note

    All electrodes numbers must be unique and non-negative. The numbers must be contiguous (0 to n) spread over however many grids.

    Default is None.

Working with Multiple Grid Layouts#

Grid layouts enable precise specification of how electrodes are arranged physically. This is especially useful for visualizing and analyzing high-density EMG recordings with multiple electrode grids:

>>> import numpy as np
>>> import matplotlib.pyplot as plt
>>> from myoverse.datatypes import EMGData, create_grid_layout
>>>
>>> # Create sample EMG data for 61 electrodes with 1000 samples each
>>> emg_data = np.random.randn(61, 1000)
>>> sampling_freq = 2048  # Hz
>>>
>>> # Create layouts for three different electrode grids
>>> # First grid: 5Ã5 array with sequential numbering (0-24)
>>> grid1 = create_grid_layout(5, 5, fill_pattern='row')
>>>
>>> # Second grid: 6Ã6 array with column-wise numbering
>>> grid2 = create_grid_layout(6, 6, fill_pattern='column')
>>> # Shift indices to start after the first grid (add 25)
>>> grid2[grid2 >= 0] += 25
>>>
>>> # Third grid: Irregular 3Ã4 array
>>> grid3 = create_grid_layout(3, 4, fill_pattern='row')
>>> grid3[grid3 >= 0] += 50
>>>
>>> # Create EMGData with all three grids
>>> emg = EMGData(emg_data, sampling_freq, grid_layouts=[grid1, grid2, grid3])
>>>
>>> # Visualize the three grid layouts
>>> for i in range(3):
...     emg.plot_grid_layout(i)
>>>
>>> # Plot the raw EMG data using the grid arrangements
>>> emg.plot('Input', scaling_factor=[15.0, 12.0, 20.0])
>>>
>>> # Access individual grid dimensions
>>> grid_dimensions = emg._get_grid_dimensions()
>>> for i, (rows, cols, electrodes) in enumerate(grid_dimensions):
...     print(f"Grid {i+1}: {rows}Ã{cols} with {electrodes} electrodes")

Plots the data for a specific representation.

Parameters:

  • representation (str) â The representation to plot.

  • nr_of_grids (Optional[int], optional) â The number of electrode grids to plot. If None and grid_layouts is provided, will use the number of grids in grid_layouts. Default is None.

  • nr_of_electrodes_per_grid (Optional[int], optional) â The number of electrodes per grid to plot. If None, will be determined from data shape or grid_layouts if available. Default is None.

  • scaling_factor (Union[float, List[float]], optional) â The scaling factor for the data. The default is 20.0. If a list is provided, the scaling factor for each grid is used.

  • use_grid_layouts (bool, optional) â Whether to use the grid_layouts for plotting. Default is True. If False, will use the nr_of_grids and nr_of_electrodes_per_grid parameters.

Examples

>>> import numpy as np
>>> from myoverse.datatypes import EMGData, create_grid_layout
>>>
>>> # Create sample EMG data (64 channels, 1000 samples)
>>> emg_data = np.random.randn(64, 1000)
>>>
>>> # Create EMGData with two 4Ã8 grids (32 electrodes each)
>>> grid1 = create_grid_layout(4, 8, 32, fill_pattern='row')
>>> grid2 = create_grid_layout(4, 8, 32, fill_pattern='row')
>>>
>>> # Adjust indices for second grid
>>> grid2[grid2 >= 0] += 32
>>>
>>> emg = EMGData(emg_data, 2000, grid_layouts=[grid1, grid2])
>>>
>>> # Plot the raw data using the grid layouts
>>> emg.plot('Input')

Plots the 2D layout of a specific electrode grid with enhanced visualization.

Parameters:

  • grid_idx (int, optional) â The index of the grid to plot. Default is 0.

  • show_indices (bool, optional) â Whether to show the electrode indices in the plot. Default is True.

  • cmap (Optional[plt.cm.ScalarMappable], optional) â Custom colormap to use for visualization. If None, a default viridis colormap is used.

  • figsize (Optional[Tuple[float, float]], optional) â Custom figure size as (width, height) in inches. If None, size is calculated based on grid dimensions. Ignored if an existing axes object is provided.

  • title (Optional[str], optional) â Custom title for the plot. If None, a default title showing grid dimensions is used.

  • colorbar (bool, optional) â Whether to show a colorbar. Default is True.

  • grid_color (str, optional) â Color of the grid lines. Default is âblackâ.

  • grid_alpha (float, optional) â Transparency of grid lines (0-1). Default is 0.7.

  • text_color (str, optional) â Color of the electrode indices text. Default is âwhiteâ.

  • text_fontsize (int, optional) â Font size for electrode indices. Default is 10.

  • text_fontweight (str, optional) â Font weight for electrode indices. Default is âboldâ.

  • highlight_electrodes (Optional[List[int]], optional) â List of electrode indices to highlight. Default is None.

  • highlight_color (str, optional) â Color to use for highlighting electrodes. Default is âredâ.

  • save_path (Optional[str], optional) â Path to save the figure. If None, figure is not saved. Default is None.

  • dpi (int, optional) â DPI for saved figure. Default is 150.

>>> import numpy as np
